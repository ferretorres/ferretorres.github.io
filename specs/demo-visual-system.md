# Demo visual system spec

## Goal

Make Ferre Torres B.V. look like an AI implementation company that builds working systems for companies, not a text-only consultancy website.

The site should show credible synthetic product surfaces for Company Brain, RAG evaluation, finance intelligence, AI PM assistance, agentic workflows, and AI-first software engineering. The visuals must support enterprise buyer trust without implying that synthetic screenshots are live client systems.

## Positioning

Primary positioning:

- AI consultancy for companies.
- AI-first transformation and implementation.
- Company Brain systems.
- RAG implementation and evaluation.
- Agentic workflows with human approvals.
- Finance intelligence dashboards and operational AI.
- AI software engineering at speed.

## Design rules

- Visuals must look operational, not decorative.
- Synthetic data must be clearly labeled.
- Each visual should explain inputs, AI processing, human control, and business output.
- Pages should keep a clear CTA: request a guided demo or start a 30-second project brief.
- No fabricated client names, metrics, certifications, or deployments.
- No claims that demo previews are live SaaS products.
- Static GitHub Pages only for this phase.

## Spec-driven cycle

Each ticket follows this loop:

1. Spec: define buyer problem, user story, content requirements, acceptance criteria, and verification.
2. Build: implement the smallest useful slice.
3. Verify: run static checks and responsive browser checks.
4. Deploy: commit and push after verification.
5. Learn: note what needs proof, screenshots, analytics, or follow-up content.

## Phases

### Phase 1: Visual credibility foundation

- Define synthetic visual language.
- Build reusable demo visual components in HTML/CSS.
- Add a demo gallery page.
- Add homepage AI operating layer visual.

### Phase 2: Demo page upgrades

- Upgrade Company Brain guided demo.
- Upgrade RAG evaluation guided demo.
- Upgrade finance intelligence guided demo.
- Upgrade AI PM assistant guided demo.
- Upgrade agentic workflow guided demo.

### Phase 3: Technical buyer depth

- Add architecture diagrams.
- Connect demo pages to technical questions.
- Add implementation pattern sections.

### Phase 4: Enterprise trust

- Add proof-style pattern pages.
- Improve CTA flow.
- Run SEO, accessibility, performance, and responsive QA.

## Required verification

- `git diff --check`
- JSON-LD parse check for all HTML pages.
- SEO metadata check for indexable HTML pages.
- Sitemap coverage check.
- Internal link check.
- Browser checks on desktop and mobile for new/changed pages.
- Live HTTPS spot checks after deployment.
