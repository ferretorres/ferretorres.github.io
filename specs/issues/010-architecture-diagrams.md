# Build: Architecture diagrams for technical buyers

## Problem

Technical buyers need enough architectural detail to trust the delivery approach before booking a call.

## User story

As a CTO or Head of Data/AI, I want to see the common implementation architecture, so I can judge whether Ferre Torres B.V. understands production constraints.

## Scope

- Add architecture diagrams for:
  - Company Brain / AI operating layer.
  - RAG implementation and evaluation.
  - Agentic workflows with approval gates.
  - Finance intelligence data flow.
- Use static HTML/CSS diagrams or lightweight inline SVG where appropriate.
- Include common components: ingestion, permissions, search/vector layer, orchestration, evaluation, monitoring, audit logs, dashboards, and human review.

## Acceptance criteria

- Architecture diagrams are readable on desktop and mobile.
- They do not require external rendering libraries.
- Diagrams link to relevant demo and solution pages.
- Technical copy stays concise and commercially useful.

## Verification

- Browser desktop and mobile checks.
- Check no text overlap.
- Internal links pass.
