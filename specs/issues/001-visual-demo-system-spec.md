# Spec: Visual demo system and synthetic data rules

## Problem

The site explains AI implementation services, but it does not yet show enough operational product surfaces. Enterprise buyers need visual proof that the company can design concrete AI systems, even when real client dashboards cannot be shown publicly.

## User story

As an enterprise buyer, I want to see credible synthetic examples of the systems Ferre Torres B.V. can build, so I can understand the delivery quality before requesting a meeting.

## Scope

- Define the visual language for synthetic demo screens.
- Define copy rules for labeling synthetic data.
- Define reusable page sections for demo previews.
- Define the required demo types and their buyer questions.
- Document what must never be claimed publicly.

## Acceptance criteria

- A versioned spec exists in the repo.
- Synthetic visuals have a consistent vocabulary and labeling rule.
- The spec covers Company Brain, RAG evaluation, finance intelligence, AI PM assistant, agentic workflows, and AI-first software engineering.
- The spec includes verification steps for static checks and browser checks.

- Confirm all future tickets can refer back to the spec.
