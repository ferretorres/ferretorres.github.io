# Build: Proof-style implementation pattern pages

## Problem

Enterprise buyers need evidence, but public named case studies may not be available yet. The site needs proof-style pages that explain delivery patterns without claiming confidential client details.

## User story

As a buyer, I want to see anonymized implementation patterns, so I can understand how the team creates value in real company contexts.

## Scope

- Add or expand pages for:
  - Regulated knowledge assistant pattern.
  - Finance reporting automation pattern.
  - Industrial intelligence pattern.
  - Project intelligence assistant pattern.
- Use anonymized language.
- Include problem, workflow, what was built, time-to-MVP range where defensible, data involved, governance, and reusable system.
- Do not name clients without explicit permission.

## Acceptance criteria

- Pattern pages increase buyer trust without fabricating proof.
- Pages link back to relevant demos and solution pages.
- Copy makes clear Ferre Torres B.V. is a team/company capability, not one person selling hours.
- CTA is clear.

## Verification

- Review for confidentiality and false claims.
- SEO metadata check.
- Browser desktop and mobile checks.
