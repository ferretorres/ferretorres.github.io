## Priority 2: Form endpoint decision

Goal: decide whether the static project brief should submit automatically or stay as email draft only.

Options:

- Stay email-only for now.
- Use Formspree, Basin, Tally, HubSpot, or another provider.

Recommended first move:

- Stay email-only until outreach starts or until a real missed lead risk appears.
- If you want automatic capture, create a Formspree endpoint for `jferre@ferretorres.eu` and send Codex the endpoint URL.

Acceptance criteria if automated:

- Endpoint receives a test submission.
- `privacy.html` names the provider and purpose.
- The homepage form still has email fallback.

## Priority 3: Analytics decision

Goal: know which pages and campaigns generate useful interest.

Options:

- Stay analytics-free.
- Use Plausible, Simple Analytics, or Fathom.
- Avoid Google Analytics unless there is a strong reason.

Recommended first move:

- Use Plausible or Simple Analytics when you start actively posting or sending outbound messages.

Acceptance criteria:

- Live pageview appears in analytics.
- Privacy page is updated.
- Events are tracked for contact clicks, project brief generation, guided demo page, and role pages.

## Priority 4: Proof inputs

Goal: make the site stronger for enterprise buyers without breaching confidentiality.

Create 2-3 anonymized proof notes using this template:

```text

```

Acceptance criteria:

- At least 2 notes have measurable outcomes.
- Each note has a confidentiality boundary.
- Codex can convert them into public-safe proof pages or private sales snippets.

## Priority 5: Team / delivery wording

Goal: decide whether collaborators stay private or become part of public credibility.

Choose one:

- Keep public wording generic: founder-led senior delivery network.
- Add anonymous capability roles.
- Add named collaborator profiles with permission.

Acceptance criteria:

- No person is named without approval.
- Public wording does not exaggerate capacity.
- Procurement-ready profiles can be shared privately if needed.

## Priority 6: Booking link

Goal: decide if prospects can book directly.

Options:

- No booking link; email/brief only.
- Cal.com.
- Calendly.
- SavvyCal.

Recommended first move:

- Keep email/brief only until inbound volume increases.
- Add booking later if prospects ask for scheduling or outreach volume grows.

Acceptance criteria if added:

- Link works on desktop and mobile.
- Privacy page is updated if the provider uses cookies or tracking.
- CTA says `Request demo or project discussion`, not generic `Book now`.

## Priority 7: Pricing and packaging

Goal: decide how much commercial detail should be public.

Decision needed:

- Keep pricing private for enterprise positioning.
- Publish package names without prices.
- Publish starting ranges.

Recommended first move:

- Keep pricing private.
- Use internal package names in proposals:
  - AI opportunity assessment
  - AI integration roadmap
  - AI MVP / PoC build
  - AI architecture review
  - production implementation
  - advisory retainer

Acceptance criteria:

- Proposal templates can be completed without public price anchoring.
- Minimum engagement size is known privately before outbound starts.

## Priority 8: LinkedIn and profile rollout

Goal: make the website visible where buyers already see you.

Do:

- Add `https://ferretorres.eu/` to LinkedIn profile.
- Add it to GitHub profile.
- Add it to email signature.
- Post from `LAUNCH_GUIDE.md`.
- Use `SALES_PLAYBOOK.md` for outreach.

Acceptance criteria:

- Public profiles point to the site.
- First 3-5 LinkedIn posts are published.
- Any inbound leads are tracked with `LEAD_INTAKE_CHECKLIST.md`.
